Jenkins — Friendly, attractive, step‑by‑step install guide

Commands and short explanations for Ubuntu/Debian, CentOS/Fedora, macOS, Windows and Docker. Copy‑buttons are included for convenience.

Quick checklist

  • Have sudo/admin access on the machine
  • Install a supported Java (OpenJDK 11 or later recommended)
  • Open port 8080 (or use reverse proxy like Nginx)

Ubuntu / Debian (recommended flow)

This installs Jenkins from the official Jenkins package repository.
sudo apt update
sudo apt install -y openjdk-11-jdk
wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo gpg --dearmor -o /usr/share/keyrings/jenkins-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/jenkins-keyring.gpg] https://pkg.jenkins.io/debian-stable binary/" | sudo tee /etc/apt/sources.list.d/jenkins.list > /dev/null
sudo apt update
sudo apt install -y jenkins
sudo systemctl enable --now jenkins

Then visit http://your_server_ip:8080 and follow the unlock & setup wizard.

RHEL / CentOS / Rocky / Fedora

Uses the rpm/dnf/yum flow with Jenkins repo.
sudo dnf install -y java-11-openjdk
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/redhat-stable/jenkins.repo
sudo rpm --import https://pkg.jenkins.io/redhat-stable/jenkins.io.key
sudo dnf install -y jenkins
sudo systemctl enable --now jenkins

Open firewall port 8080 if using firewalld: sudo firewall-cmd --permanent --add-port=8080/tcp && sudo firewall-cmd --reload

macOS (Homebrew) & Docker

brew update
brew install jenkins-lts
brew services start jenkins-lts

Or run via Docker (good for isolation):

docker run -p 8080:8080 -p 50000:50000 --name jenkins-lts jenkins/jenkins:lts

Windows (MSI installer)

Download the Windows installer (.msi) from the official Jenkins download page and run it. Use the setup wizard to configure service user and port.

After installer completes, open http://localhost:8080 and follow the unlock wizard.

Post‑install — unlock & finish setup

When you visit Jenkins for the first time it asks for an initial admin password stored on the server.
sudo cat /var/lib/jenkins/secrets/initialAdminPassword

Paste that value into the web setup wizard, install suggested plugins and create your first admin user.

Secure Jenkins (recommended)

  • Run Jenkins behind Nginx as a reverse proxy with SSL (Let's Encrypt recommended).
  • Disable anonymous access and integrate with LDAP/SSO if needed.
  • Keep Jenkins and plugins updated.

Useful commands

sudo systemctl status jenkins
sudo journalctl -u jenkins -f
sudo apt remove --purge jenkins

Troubleshooting tips

  • If Jenkins doesn't start, check Java version with java -version.
  • Check logs: /var/log/jenkins/jenkins.log or use journalctl.
  • On permission errors, ensure the jenkins user can access required directories.
This single‑file guide is meant for learning and quick setup. For production read the official Jenkins documentation and security guides.
